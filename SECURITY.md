# Security Policy

## Supported versions

Security fixes are applied to the latest release branch and `main`.

## Reporting a vulnerability

Please do **not** open public issues for potential vulnerabilities.

Instead, report privately by email:

- `security@oceandl.dev` (placeholder, replace with real maintainer address)

Include:

- Affected version/commit
- Reproduction steps or proof of concept
- Expected vs. actual behavior
- Potential impact

We will acknowledge reports as quickly as possible and coordinate disclosure after a fix is ready.
